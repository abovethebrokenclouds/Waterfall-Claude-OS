GOAL
Bring the Auto-Shorts app to feature parity with the canonical backend: every
per-short AI "skill" should be reachable from the UI. Implement each as a
Lovable Cloud edge function backed by Lovable AI, wire a typed caller in
src/lib/api.ts (each with a graceful fallback on error), and surface it in the
ShortCard or EditModal. Keep all existing behaviour (generate, variation, demo
fallback) intact.

Every edge function takes JSON, calls Lovable AI with a strict "return ONLY
JSON" instruction, validates/clamps the result, and returns the shape below.
If the model errors or returns junk, return a small deterministic fallback so
the UI never breaks.

----------------------------------------------------------------------------
EDGE FUNCTIONS (one per skill)
----------------------------------------------------------------------------

1) hook-variations
   In:  { plan, count?: number (default 3, max 8) }
   Out: { hooks: string[] }  — `count` distinct hooks (<=12 words), varied
        angle (curiosity, bold claim, question, stat, contrarian).

2) virality-score
   In:  { plan }
   Out: { score: 0-100,
          breakdown: { hook:0-100, pacing:0-100, payoff:0-100, shareability:0-100 },
          reasons: string[] (<=5) }  — clamp every number to 0-100.

3) cover-concept
   In:  { plan, brandColor?: string }
   Out: { coverText: string(<=5 words), subText, backgroundIdea, emoji,
          textColor: "#RRGGBB" }  — invalid hex -> brandColor (if valid) else "#FFFFFF".

4) optimize-title  (SEO title variants)
   In:  { plan, platform?: "tiktok"|"instagram_reels"|"youtube_shorts"|"facebook_reels"|"x" }
   Out: { titles: string[] (3-6, de-duplicated, platform-aware length),
          keywords: string[] (search terms to work in) }

5) cta  (call-to-action optimizer)
   In:  { plan, platform?, goal?: string (e.g. "follows","saves","link clicks") }
   Out: { ctas: string[] }  — up to 5 stronger CTAs (<=10 words), de-duplicated.

6) caption-emphasis  (karaoke word highlighting)
   In:  { text: string }
   Out: { words: { word: string, emphasize: boolean }[] }
   NOTE: tokenize `text` on whitespace in the edge function and return ONE entry
   per token in original order. Ask the model only for which words to emphasize
   (the 2-4 punchiest keywords); mark those true, the rest false. Blank text -> [].

7) hashtag-strategy
   In:  { plan, platform? }
   Out: { broad: string[](<=5), niche: string[](<=8), branded: string[](<=3) }
        — strip leading '#', lowercase, de-duplicate across tiers.

8) broll-suggestions
   In:  { plan, transcriptExcerpt?: string, count?: number (default 4) }
   Out: { suggestions: { atSec: number, idea: string, keyword: string }[] }
        — clamp atSec to [0, plan.durationSec]; drop empty ideas.

9) series-plan  (package multiple shorts into a numbered series)
   In:  { plans: ShortPlan[], topic?: string }
   Out: { seriesTitle, parts: { shortId, order, partTitle, teaser }[], cadence }
        — parts reference only real plan ids, ordered 1..N. Empty plans -> empty parts.

10) retention  (predicted drop-off curve + fixes)
    In:  { plan, transcriptExcerpt?: string }
    Out: { score: 0-100, dropoffs: { atSec: number, risk: "low"|"medium"|"high",
           fix: string }[] }  — clamp atSec to [0, plan.durationSec], coerce risk,
           drop empty fixes, max 5.

11) engagement-prompt  (comment-driving pinned questions)
    In:  { plan, platform? }
    Out: { prompts: string[] }  — up to 5 (<=15 words), de-duplicated; vary angle
           (poll, hot take, fill-in-the-blank, tag-a-friend, question).

12) music  (audio direction)
    In:  { plan }
    Out: { mood: string, genres: string[](<=4), tempo: "slow"|"medium"|"fast",
           searchTerms: string[](<=5) }  — coerce tempo, de-duplicate lists.

13) variation  (re-angle one short)  — keep your existing one if present.
    In:  { plan, instruction: string }
    Out: a patched plan { title, hook, theme, cta, layout }.

----------------------------------------------------------------------------
FRONTEND WIRING (src/lib/api.ts)
----------------------------------------------------------------------------
Add a typed caller per function via functions.invoke, each returning the typed
shape above and falling back gracefully (empty/derived value) on error so the
UI degrades instead of throwing.

----------------------------------------------------------------------------
UI SURFACES
----------------------------------------------------------------------------
ShortCard (compact, lazy — fetch after grid render or on tap):
  - VIRALITY badge "🔥 82", coloured by band: >=75 green, 50-74 amber, <50 slate.
  - A small "More" / toolkit affordance opening the EditModal on the right tab.

EditModal — tabs/sections, each a button that calls its function and renders
results inline; applying a result updates the short:
  a) A/B Hooks      — list hooks; tap one to set the short's hook.
  b) Title (SEO)    — list title variants (tap to apply) + a keywords chip row.
  c) CTA            — list CTA variants; tap to set the short's cta.
  d) Caption        — render the caption with emphasized words bolded/coloured
                      (karaoke preview) from caption-emphasis; "Copy caption".
  e) Hashtags       — three labelled rows (broad / niche / branded) + "Copy all".
  f) B-roll         — timeline list of { atSec, idea, keyword }; "Copy shotlist".
  g) Cover          — 9:16 preview tile (backgroundIdea caption, big coverText in
                      textColor, small subText, emoji) + "Copy cover text".
  h) Score          — score + breakdown bars + reasons list.
  i) Retention      — overall retention score + a list of drop-off points
                      (atSec, risk pill, fix); colour the risk pill high/med/low.
  j) Engagement     — list pinned-comment prompts; "Copy" each.
  k) Music          — mood line, tempo pill, genre chips, and search-term chips
                      (each chip copies a trending-audio search query).

Series (grid-level action, optional):
  - A "Make Series" button over the whole grid that calls series-plan with all
    shorts and shows the seriesTitle, ordered parts (partTitle + teaser), and
    cadence; "Copy series plan".

----------------------------------------------------------------------------
ACCEPTANCE
----------------------------------------------------------------------------
- Every skill above is reachable from the UI and returns real AI output.
- Each card shows a virality badge; the edit modal exposes hooks, title, CTA,
  caption karaoke, hashtags, b-roll, cover, and score.
- A grid-level "Make Series" produces a numbered plan.
- No external backend, API key, or hosting — all via Lovable Cloud + Lovable AI.
- Existing generate / variation / demo-fallback behaviour still works.
