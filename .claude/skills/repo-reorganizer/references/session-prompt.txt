THE ARCHITECTURE STANDARD (summary)
══════════════════════════════════════════════════════════════

Every Waterfall APP repo is normalized to 16 logical modules:
  api · agents · tools · workflows · memory · lib · ui · env · config
  scripts · docs · tests · platform · integrations · skills · registry

For framework apps (TanStack/Vite/Lovable) the taxonomy maps UNDER src/:
  src/api  src/agents  src/tools  src/lib  src/ui  etc.
  App-specific logic → src/platform/<app-slug>/
  Framework dirs stay where the framework expects them (src/routes, supabase/, public/)

Waterfall-Claude-OS is EXEMPT — it is a registry/control-plane, not an app.
Do NOT apply the app folder template to it.

THE ONE RULE: every AI call routes through the Waterfall Super Agent.
Never a raw fetch to a model API, hardcoded model string, or manual max_tokens.
